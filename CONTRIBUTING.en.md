# Contributing

[中文](CONTRIBUTING.md) | **English**

Thanks for your interest in this project!

## Development Environment
- Python 3.11
- Install dependencies: `pip install -r requirements-dev.txt` (includes pytest)
- Copy config: `cp .env.example .env` (runs in Mock mode without Azure credentials)

## Run Locally
```bash
python scripts/seed_data.py          # generate the sample knowledge base
uvicorn app.main:app --reload --port 8000
```

## Pre-commit Self-checks
- Run unit tests: `pytest -q` (48 tests, isolated via a temp directory, no server needed)
- Run the end-to-end test after starting the server: `python scripts/smoke_test.py`
- Make sure NOT to commit the following (already ignored in .gitignore):
  - `.env` (real credentials)
  - `data/chroma/`, `data/app.db` (runtime data)
  - `data/knowledge/` (generated by the seed script, or customer-private documents)

## pre-commit Hooks (recommended)
This repo ships a `.pre-commit-config.yaml` that runs automatically before each commit:
trailing-whitespace cleanup, YAML/JSON validation, private-key detection, and pytest unit tests.

```bash
pip install pre-commit
pre-commit install            # install the git hook
pre-commit run --all-files    # optional: run once across all files
```
Once installed, `git commit` triggers the checks automatically; any failing check blocks the commit.
See [docs/pre-commit.en.md](docs/pre-commit.en.md) for details.

## Code Style
- Backend follows PEP 8; keep functions small and clear.
- Use concise imperative commit messages (English or Chinese).

## Security
- Never commit any secrets, passwords, or real customer bid data.
- Change `AUTH_SECRET` in `.env` for production deployments.
